### Step 2 — Proxy ETag / conditional GET on the MinIO routes *(plan §10.2 (4) recommendation; deferred from M2 Step 3)* — after Step 1 (file-collision serialization only)

**Files:** `dashboard/server/src/minio-client.js`, `dashboard/server/src/handler.js`, `dashboard/server/test/minio-client.test.js`, `dashboard/server/test/handler.test.js`, `dashboard/README.md`.
**Seams:**
- `handleMinio`'s object path drops all upstream headers today: `res.writeHead(200, {'content-type': guessContentType(key)})` (`handler.js:94-99`) — MinIO's native `ETag`/`Last-Modified` (which the objects DO carry, §10.2 (4)) never reach the browser, so every 15 s poll re-downloads every `meta.json`/`plan.md`.
- `MinioClient._request(method, path, query, {asBuffer})` (`minio-client.js:34-73`) has no header pass-through; `getObject` at `:79-82`.
- SigV4 signs only `host`/`x-amz-content-sha256`/`x-amz-date` (`sigv4.js:73-80` — `SignedHeaders` is built from that fixed map), so forwarding `If-None-Match`/`If-Modified-Since` as **unsigned** request headers does not break the signature. ⚠️ MinIO honoring unsigned conditional headers on GetObject is standard S3 behavior but is **not verifiable from this checkout** — stub-tested only (ledger #1).
**Change:** `getObject(key, {conditionalHeaders})` merges the caller's `if-none-match`/`if-modified-since` into the transport request (NOT into the signature map); `handleMinio` forwards those two headers from the browser request; on upstream **304** relay a bodyless 304; on **200** relay `etag` + `last-modified` + `Cache-Control: no-cache` (forces revalidation instead of heuristic caching — the SPA polls fixed URLs, so the browser's HTTP cache supplies `If-None-Match` automatically; **no SPA change**). Listing responses (`handler.js:105-122`) stay uncached (they aggregate many objects). Controller-proxied GETs stay fetch-fresh (small JSON; §10.2 (4) explicitly says don't bother).
**Tests:** minio-client — conditional headers sent unsigned, signature unchanged (assert the `SignedHeaders` string is byte-identical with/without them); handler — 200 relays etag/last-modified/no-cache, request `If-None-Match` forwarded, upstream 304 → bodyless 304, listing path unaffected, the 404-fallback-to-listing behavior unchanged (`handler.js:100-107`).
**Acceptance:** `npm test` green; `docker build` green; README notes the caching contract. **Deferred-to-deploy:** real-MinIO 304 behavior + observed bandwidth drop (ledger #1/#2).
